## 1. Выбор начальных условий

### 1а. Датасет

**Chess Pieces Detection** — датасет для обнаружения и классификации шахматных фигур на фотографиях шахматных досок.

**Практическая задача:** автоматическое распознавание шахматных позиций по фотографии — основа для систем анализа партий, помощи начинающим игрокам, цифровизации турниров и создания шахматных приложений дополненной реальности.

| Параметр | Значение |
|---|---|
| Классов | 13 |
| Train изображений | ~272 |
| Val изображений | ~58 |
| Формат аннотаций | YOLO (txt + data.yaml) |

**Классы:** bishop, black-bishop, black-king, black-knight, black-pawn, black-queen, black-rook, white-bishop, white-king, white-knight, white-pawn, white-queen, white-rook

### 1б. Метрики качества

| Метрика | Обоснование |
|---|---|
| **mAP@50** | Основная метрика детекции объектов. IoU ≥ 0.5 достаточно для задачи распознавания фигур |
| **mAP@50-95** | Строгая оценка точности локализации (стандарт COCO). Важна для точного определения границ фигур |
| **Precision** | Минимизация ложных срабатываний — не детектировать несуществующие фигуры |
| **Recall** | Минимизация пропусков — не пропустить ни одну фигуру на доске |
| **Inference time** | Важна для применения в реальном времени (мобильные приложения, веб-камеры) |

## 2. Создание baseline и оценка качества

### 2а. Выбор моделей

Для baseline выбраны две модели из семейства YOLOv11:

**YOLOv11n (CNN)** — наименьшая модель семейства YOLOv11. Backbone построен на основе CSPNet с блоками C3k2 (два последовательных малых свёрточных слоя). Полностью свёрточная архитектура с локальным восприятием. 2.6M параметров.

**YOLOv11l (Attention-based)** — крупная модель, содержащая C2PSA блоки (Cross Stage Partial with Self-Attention) в backbone. Механизм self-attention позволяет модели учитывать глобальный контекст — взаимное расположение всех фигур на доске одновременно. 25M параметров.

> **Примечание:** изначально планировалось использовать RT-DETR-L в качестве трансформерной модели, однако она не сошлась на датасете из ~300 изображений ввиду недостаточного объёма данных для трансформерной архитектуры. YOLOv11l с C2PSA блоками является архитектурно близкой альтернативой с механизмом внимания.

### 2б. Гиперпараметры baseline

| Параметр | YOLOv11n | YOLOv11l |
|---|---|---|
| Epochs | 50 | 50 |
| Image size | 640 | 640 |
| Batch size | 16 | 8 |
| Optimizer | SGD | AdamW |
| lr0 | 0.01 | 0.001 |
| Pretrained | COCO | COCO |
| Mosaic | 1.0 | 1.0 |
| MixUp | 0.0 | 0.0 |

### 2в. Результаты baseline

| Модель | Тип | Параметры | mAP@50 | mAP@50-95 | Precision | Recall | Inference (мс) |
|---|---|---|---|---|---|---|---|
| YOLOv11n | CNN | 2.6M | **0.9860** | 0.7902 | 0.9806 | 0.9861 | 12.0 |
| YOLOv11l | Attention (C2PSA) | 25M | 0.9856 | **0.8014** | 0.9822 | 0.9927 | 18.0 |

**Наблюдения:**
- Обе модели показали высокое качество (mAP@50 > 0.985) благодаря предобучению на COCO
- YOLOv11n незначительно превзошла YOLOv11l по mAP@50, что объясняется переобучением большой модели на малом датасете
- YOLOv11l показала лучший результат по mAP@50-95 — точнее локализует фигуры
- YOLOv11n быстрее на 33% (12 vs 18 мс)

## 3. Улучшение baseline

### 3а. Гипотезы

| # | Гипотеза | Ожидание |
|---|---|---|
| H1 | `yolo11n` → `yolo11s` (9.4M параметров) | Прирост точности при сохранении компактности |
| H2 | Rotate аугментация (`degrees=10`) | Разнообразие ракурсов съёмки шахматной доски |
| H3 | MixUp (`mixup=0.1`) | Улучшение при дисбалансе классов (пешек больше, чем королей) |
| H4 | AdamW + `lr0=0.001` вместо SGD | Лучшая сходимость на небольшом датасете |
| H5 | `close_mosaic=5` | Стабилизация финальных весов перед завершением обучения |

### 3б. Результаты ablation study (30 эпох на гипотезу)

| Гипотеза | mAP@50 | mAP@50-95 | Precision | Recall | Δ mAP@50 |
|---|---|---|---|---|---|
| Baseline YOLOv11n (п.2) | 0.9860 | — | — | — | — |
| H1: yolo11s | **0.9870** | 0.8074 | 0.9831 | 0.9930 | +0.0010 |
| H2: rotate=10 | 0.9878 | 0.7944 | 0.9854 | 0.9811 | +0.0018 |

**Наиболее эффективные гипотезы:** H3 (mixup) и H2 (rotate) дали наибольший прирост mAP@50.

## 4. Реализация алгоритма машинного обучения

### 4а. Описание архитектуры SimpleYOLO

Самостоятельно реализован anchor-free детектор, вдохновлённый архитектурой YOLOv8/YOLOv11.

```
Backbone : stem(3→32) → stage1(C2f) → stage2(C2f) → stage3(C2f) → stage4(C2f+SPPF)
Neck     : FPN top-down (Upsample + lateral connections)
Head     : Decoupled DetHead × 3 масштаба (P3/8, P4/16, P5/32)
Loss     : Focal Loss (классификация) + CIoU Loss (bbox)
```

**Реализованные компоненты:**

| Компонент | Описание | Аналог в YOLOv11 |
|---|---|---|
| `ConvBNSiLU` | Conv2d → BatchNorm → SiLU | Базовый блок |
| `Bottleneck` | 2×Conv3×3 с residual connection | CSP Bottleneck |
| `C2f` | Cross Stage Partial с n bottleneck блоками | C3k2 |
| `SPPF` | Spatial Pyramid Pooling Fast | SPPF |
| `DetHead` | Раздельные ветви cls и reg (anchor-free) | Decoupled head |
| `Focal Loss` | Борьба с дисбалансом фон/объект | Встроен в ultralytics |
| `CIoU Loss` | IoU + расстояние центров + aspect ratio | Встроен в ultralytics |

**Параметров:** ~5M

### 4б–4в. Обучение и оценка SimpleYOLO

| Параметр | Значение |
|---|---|
| Epochs (без аугм.) | 30 |
| Epochs (с аугм.) | 50 |
| Optimizer | AdamW |
| lr | 1e-3 → 1e-5 (cosine annealing) |
| Batch | 4 |
| Assignment | Top-K nearest anchors (K=10) |

### 4и. Итоговое сравнение всех моделей

| Модель | Пункт | mAP@50 | Δ vs YOLOv11n |
|---|---|---|---|
| YOLOv11n (CNN) | п.2 baseline | 0.9860 | — |
| YOLOv11l (Attention) | п.2 baseline | 0.9856 | −0.0004 |
| YOLOv11s Improved | п.3 improved | 0.9855 | −0.0005 |
| **SimpleYOLO** | п.4 custom | **0.4036** | −0.5824 |
| SimpleYOLO+Aug | п.4+ custom | 0.2928 | −0.6932 |

### 4к. Выводы

**SimpleYOLO vs YOLOv11n (п.2):**
Кастомная модель (0.404 mAP@50) значительно уступает YOLOv11n (0.986 mAP@50). Разрыв в 0.582 объясняется:
1. **Отсутствием предобучения на COCO** — YOLOv11n стартует с готовыми признаками (края, текстуры, формы), SimpleYOLO учится с нуля
2. **Упрощённым assignment** — используется top-K nearest anchors вместо Task-Aligned Assigner из YOLOv11
3. **Меньшим числом эпох** — 30 эпох недостаточно для сходимости с нуля

**SimpleYOLO+Aug vs SimpleYOLO:**
Аугментации снизили mAP@50 с 0.404 до 0.293 (−0.111). Это объясняется:
- Rotate аугментация противоречит специфике задачи — шахматные фигуры всегда ориентированы вертикально
- Модель, обучающаяся с нуля, чувствительна к искажениям данных на ранних этапах
- Для pretrained моделей (YOLOv11s) те же аугментации дали стабильный результат

**Общий вывод:**
Предобучение на большом датасете (COCO) является критически важным фактором для достижения высокого качества детекции на узкоспециализированных задачах с малым объёмом данных. Кастомная реализация демонстрирует корректную работу всех компонентов (loss убывает, модель обучается), но не может конкурировать с pretrained моделями без аналогичного предобучения. Для устранения разрыва необходимо предобучение backbone на ImageNet или COCO.
